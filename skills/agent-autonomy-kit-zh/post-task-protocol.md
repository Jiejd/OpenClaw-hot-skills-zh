# 任务后协议

**状态：** 必须执行 — 每次任务完成后运行  
**版本：** 1.0  
**最后更新：** 2026 年 2 月 4 日

---

## 规则

**完成任何任务（直接工作或 subagent 返回）后，你必须在询问人类输入之前再次检查队列。**

---

## 为什么需要这个

**问题：** 智能体在心跳开始时检查队列（✅）但任务完成后不检查（❌）。这造成了一个缺口，自主执行在此处断裂：

```
心跳 → 检查队列 → Spawn 智能体 → 任务完成 → ❌ "下一步是什么？"
```

**解决方案：** 闭合回路。每次任务完成都是一次微型心跳：

```
心跳 → 检查队列 → 工作 → 完成 → 再次检查队列 → 回路
```

---

## 协议

### 步骤 1：更新队列状态

将已完成的任务从"Ready"或"In Progress"移到"Done"：

```markdown
## ✅ Done Today
- [x] Colony Skill 集成（下午 5:00 完成）
```

### 步骤 2：运行队列检查器（必须执行）

```bash
node skills/agent-autonomy-kit/check-queue.js
```

**每次完成任务后都要执行。** 没有例外。

### 步骤 3：解读退出码

**退出码 1（存在 HIGH/CRITICAL 任务）：**
- 从输出中读取最高优先级任务
- 立即为该任务 spawn 智能体
- 记录到每日记忆
- **不要问人类"我们接下来该处理什么？"**

**退出码 0（队列清空或仅 LOW 任务）：**
- 检查 5D 循环阶段（读取 HEARTBEAT.md）
- 发现过时（>2 小时）？ → Spawn Scout
- 有未分类的机会？ → Spawn Sage
- 有已设计好待构建的项目？ → Spawn Link
- 如果所有阶段空闲：检查 heartbeat-state.json 中的计划任务
- 如果确实无事可做：然后你可以说 HEARTBEAT_OK 或询问人类

### 步骤 4：记录决策

写入 `memory/YYYY-MM-DD.md`：

```markdown
16:05 - 完成 Colony Skill 集成
16:05 - 运行队列检查器：退出码 1（仍有 14 个 HIGH 任务）
16:06 - 最高优先级：Observability Control Plane 定位
16:06 - Spawn：echo-observability-positioning
```

## 示例

### ✅ 正确：回路到队列

**场景：** 刚完成 Colony Skill 集成

```
16:00 - Colony Skill 任务完成
16:00 - 更新 tasks/QUEUE.md（移到 Done Today）
16:00 - 运行：node skills/agent-autonomy-kit/check-queue.js
16:00 - 输出："🔴 HIGH priority tasks: 14"
16:00 - 最高优先级："Observability Control Plane 定位"
16:01 - Spawn：echo-observability-positioning，附 DEFINE 规格
16:02 - 记录到 memory/2026-02-04.md
```

**结果：** 自主执行继续。无需许可。

---

### ✅ 正确：队列清空 → 检查 5D 循环

**场景：** 完成了最后一个 HIGH 优先级任务

```
18:30 - 任务完成，队列已更新
18:30 - 运行队列检查器：退出码 0（无 HIGH 任务）
18:30 - 检查 5D 循环阶段：
  - 发现：上次运行 3 小时前（过时）
  - DEFINE：2 个机会需要规格
18:31 - 决策：Spawn Scout 进行发现
18:31 - Spawn：scout-discovery-run-13
```

**结果：** 无需请求许可，转向生成性工作。

---

### ✅ 正确：一切空闲 → 检查日程

**场景：** 队列清空，5D 阶段健康

```
22:45 - 任务完成，队列清空
22:45 - 队列检查器：退出码 0
22:45 - 5D 循环：所有阶段最新
22:45 - 检查 heartbeat-state.json：
  - 计划：Observability 发布 2 月 6 日上午 9 点 PST
  - 计划：LangChain 网络研讨会 2 月 5 日
22:46 - 决策：存在计划中的工作，现在无需操作
22:46 - 记录："任务完成，下一个计划工作明天上午 9 点"
```

**结果：** 识别了计划工作，没有请求许可。

---

### ❌ 错误：未检查队列就询问人类

**场景：** 完成任务后立即询问人类

```
16:00 - Colony Skill 任务完成
16:00 - 更新队列
16:01 - 发送："任务完成！我们接下来该处理什么？🤔"
```

**为什么错误：**
- 没有运行队列检查器
- 有 14 个 HIGH 优先级任务在等待
- 在存在自主执行路径时请求了许可

**正确操作：** 运行队列检查器，看到 14 个 HIGH 任务，spawn 下一个智能体。

---

### ❌ 错误：Subagent 返回 → 推迟决策

**场景：** Subagent 完成，主智能体推迟

```
智能体 spawn echo-memory-wars 撰写活动内容
Echo 完成："88KB 内容已就绪，所有交付物完成"
主智能体："Echo 完成了！你想什么时候发布？"
```

**为什么错误：**
- heartbeat-state.json 中存在发布日程（2 月 6 日上午 9 点）
- 询问前没有检查日程
- 推迟了一个已经做出的决策

**正确操作：** 检查日程，看到 2 月 6 日发布计划，在日志中确认日程，继续其他工作。

## 何时可以询问人类

你可以在以下情况下向人类请求输入：

1. **确实被阻塞：**
   - 缺少你无法获取的信息（API 密钥、密码、审批）
   - 外部依赖未解决（等待部署、访问权限）
   - 遇到无法解决的技术阻塞（你无法调试的错误）

2. **需要战略决策：**
   - 产品方向变更
   - 大额预算分配
   - 策略/流程变更
   - 任何影响下游人类的事项

3. **需要澄清：**
   - 队列中的任务有歧义
   - 优先级冲突（两个 CRITICAL 任务，不清楚先做哪个）
   - 范围不确定（不清楚任务应该是 HIGH 还是 LOW）

**在这些情况下：** 先记录你检查了什么，然后询问。

**示例：**
```
"已检查队列（退出码 1），最高优先级任务是'部署到生产环境。'
我需要在生产部署前获得审批。
你批准后我就可以部署。"
```

**而不是：**
```
"我们接下来该做什么？"
```

## 测试协议

### 任务完成后自检

问自己：
1. ✅ 我更新了队列吗？
2. ✅ 我运行了 `check-queue.js` 吗？
3. ✅ 我正确解读了退出码吗？
4. ✅ 我 spawn 了下一个智能体或检查了 5D 循环吗？
5. ✅ 我记录了决策吗？

如果全部 ✅ → 协议已正确遵循。

如果有任何 ❌ → 回顾协议并重试。

## 成功指标

**使用此协议之前：**
- "我们接下来该处理什么？" = 任务完成后很常见
- 请求许可 = 默认行为

**使用此协议之后：**
- "我们接下来该处理什么？" = 仅在确实卡住时
- 自主回路 = 默认行为

**测试：** 你在队列有答案时请求了许可吗？
- 如果否：✅ 协议有效
- 如果是：❌ 回顾协议，先检查队列

## 常见问题

### 问：如果任务是 LOW 优先级怎么办？

**答：** 仍然运行协议。队列中可能在你开始 LOW 任务时添加了 HIGH 优先级任务。

### 问：如果我不在心跳会话中怎么办？

**答：** 这适用于所有任务完成，无论会话类型：
- 直接人类请求 → 完成 → 检查队列
- 心跳 spawn 的工作 → 完成 → 检查队列
- Subagent 返回 → 检查队列
- Cron 任务完成 → 检查队列

### 问：如果 check-queue.js 运行失败怎么办？

**答：** 这是一个阻塞。报告："队列检查器失败，错误 X。需要帮助调试。"这是一个合理的询问。

### 问：我是否在每条消息后都检查队列？

**答：** 不。只在**完成任务后**。对话回复不需要队列检查。

**需要队列检查：**
- ✅ 完成功能构建
- ✅ 完成调研任务
- ✅ Subagent 报告完成
- ✅ 发布内容
- ✅ 部署代码

**不需要队列检查：**
- ❌ 回答事实性问题
- ❌ 确认收到消息
- ❌ 任务中途提供状态更新
- ❌ 报告阻塞
