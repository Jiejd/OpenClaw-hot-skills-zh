# Opencode 控制器

## 核心规则

Clawdbot 不编写代码。
所有规划和编码工作均在 Opencode 中完成。

## 预检

- 询问用户要使用哪个 AI 提供商。
- 询问提供商应如何进行身份验证。
- 未经用户确认，不得继续操作。

## 会话管理

- 启动 Opencode。
- 使用以下命令打开会话选择器：
  /sessions
- 如果当前项目已存在：
  - 选择已有会话。
- 未经用户批准，不得创建新会话。

## 智能体（模式）控制

- 使用以下命令打开智能体选择器：
  /agents
- 可用智能体：
  - Plan（规划）
  - Build（构建）
- 始终先选择 Plan 模式。
- 需要时使用 `/agents` 切换智能体。

## 模型选择

- 使用以下命令打开模型选择器：
  /models
- 选择用户请求的提供商。
- 如果需要身份验证：
  - 复制 Opencode 提供的登录链接。
  - 将链接发送给用户。
  - 等待用户确认后再继续。

## Plan 智能体行为

- 要求 Opencode 分析任务。
- 请求生成清晰的分步计划。
- 允许 Opencode 提出澄清问题。
- 仔细审查计划。
- 如果计划不正确或不完整：
  - 要求 Opencode 修改计划。
- 在 Plan 模式下不允许生成代码。

## Build 智能体行为

- 使用 `/agents` 切换到 Build 模式。
- 要求 Opencode 执行已批准的计划。
- 如果 Opencode 提出任何问题：
  - 立即切换回 Plan 模式。
  - 回答问题并确认计划。
  - 再切换回 Build 模式。

## 完成标准

- 重复 Plan → Build 循环，直到满足所有用户需求。
- 永远不要跳过 Plan 阶段。
- 永远不要在 Build 模式下回答问题。

## 输出格式

- 明确显示所有斜杠命令。
- 说明选择了哪个选项。
- 原样提供登录链接。
